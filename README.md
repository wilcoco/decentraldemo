# 아고라 라이브 (Agora Live)

**투표를 대체하는 실시간 민주주의 의사결정 시스템** — 데모 구현

## 문제의식

임기제 선출 투표는 "왕을 투표로 뽑는" 것과 같은 구조적 문제를 가진다.
선출 시점의 여론이 임기 내내 고정되는 것은 시스템에 **레그(lag)** 를 심는 것이며,
이 레그는 포퓰리즘·책임 회피·민의와 권력의 괴리를 낳는다.

이 프로젝트는 다음 원칙으로 레그를 제거한다:

1. **임기 없음** — 지지는 언제든 즉시 철회할 수 있고, 모든 의견의 지위는
   저장되지 않고 매 순간 현재 지지·검증 상태로부터 다시 계산된다.
2. **과제 분할** — 국정 과제를 분야(복지·경제·환경 등)별 의제로 분할해,
   시민이 사안 단위로 참여한다. 단일 대표에게 모든 권한을 몰아주지 않는다.
3. **의견의 발현과 검증** — 단순 찬반 투표가 아니다. 시민이 사안에 대해
   다양한 의견(주장)을 제안하고, 좋은 의견은 실시간 지지를 얻어 부상하며,
   근거 제출과 반론 제기를 통해 다양한 주체에 의해 검증된다.
   반론이 우세한 의견은 지지가 높아도 채택되지 않는다.
4. **분야별 위임 (액체 민주주의)** — 직접 참여하지 않는 분야는 신뢰하는
   시민에게 목소리를 위임할 수 있다. 위임은 연쇄적으로 흐르고(순환은 안전하게
   차단), **직접 참여가 항상 위임을 우선**하며, 위임은 즉시 회수할 수 있다.
5. **신뢰 네트워크 (자체 블록체인)** — 모든 제안·지지·철회·위임·검증 행위는
   시민의 ECDSA 개인키로 서명되어 해시 체인 블록에 기록된다.
   누구나 체인 전체(해시 연결·작업 증명·서명)를 재검증할 수 있어,
   조작 불가능한 감사 기록이 된다.

## 의견의 실시간 지위

| 지위 | 조건 |
|------|------|
| 제안됨 | 검증 활동 없음 |
| 검증중 | 근거 또는 반론이 존재 |
| 우세 | 유효 지지 가중치가 전체 시민의 25% 이상 |
| **채택** | 유효 지지 50% 이상 **그리고** 검증됨(근거 ≥ 반론) |
| 반박됨 | 반론이 근거보다 많음 (지지와 무관하게 채택 불가) |

지위는 어디에도 저장되지 않는다. 지지가 빠지면 그 순간 채택 지위도 사라진다.
이것이 "레그 없는 민주주의"의 핵심이다.

## 실행

의존성이 전혀 없다. Node.js 18+ 만 있으면 된다.

```bash
npm start        # http://localhost:3000
npm test         # 핵심 로직 테스트
```

웹 UI에서:
- 시민을 선택(또는 새로 등록)하고 의제별로 의견 제안 / 지지·철회 / 근거·반론 제출
- 분야별 위임을 설정·회수 (오른쪽 패널)
- **시뮬레이션 시작** 버튼을 누르면 가상 시민들의 활동으로 여론이 실시간으로
  움직이는 것을 볼 수 있다 (SSE로 모든 접속자에게 즉시 반영)
- 오른쪽 하단에서 신뢰 체인의 최근 블록과 무결성 상태를 확인

## 아키텍처

```
src/blockchain.js   신뢰 네트워크: ECDSA 서명 트랜잭션 + 해시 체인 + 작업 증명 + 전체 검증
src/democracy.js    핵심 로직: 시민/의제/의견, 실시간 지지, 위임 가중치 해석, 지위 계산
src/seed.js         데모 초기 데이터 (국정 과제 6건, 시민 12명, 의견·위임·검증 활동)
src/server.js       의존성 없는 HTTP 서버: JSON API + SSE 실시간 스트림 + 시뮬레이션
public/             한국어 웹 UI (실시간 갱신)
test/test.js        체인 무결성·변조 감지·위임 해석·지위 변동 테스트
```

### 위임 가중치 해석 규칙

의제 단위로 계산한다:
- 해당 의제의 어떤 의견에든 **직접 지지한 시민은 자기 목소리를 쓴다** (가중치 1).
- 직접 행동하지 않은 시민의 표는 그 의제 분야의 위임 사슬을 따라 흐른다.
- 사슬의 끝이 직접 지지자면 그의 가중치에 +1, 아니면 (미참여·순환) 그 표는 계산에서 제외.
- 의견의 유효 지지 = 그 의견의 직접 지지자들의 가중치 합.

## 2단계: 블록체인을 넘어 — 위브(Weave) 프로토콜

`src/weave/`는 분권화의 다음 단계다. 전역 블록체인을 버리고
**"시민별 서명 로그 + 관심 기반 복제 + 상호참조"** 구조를 택했다.

### 왜 블록체인이 아닌가

블록체인의 경쟁적 체인 연장(채굴/합의)은 **이중지불을 막기 위한 전역 단일
순서**가 필요해서 존재한다. 그러나 의견 표명에는 이중지불 문제가 없다 —
필요한 것은 "각 시민의 최신 상태"뿐이다. 따라서:

- **전역 체인 없음**: 시민마다 자기만의 서명 로그(개인 해시 체인)를 가진다.
  행위 1건의 비용 = 서명 1개. 채굴도 수수료도 없다.
- **전역 합의 없음**: 집계는 (작성자, 순번) 기준 최신 우선 병합(CRDT)이라
  항목이 어떤 순서로 도착해도 같은 결과에 수렴한다. 어느 노드가 계산해도 동일하다.
- **전체 복제 강제 없음**: 각 노드는 자기가 관심 있는 주제만 복제·저장한다.
  모든 사람이 신뢰 네트워크 유지에 리소스를 쏟을 필요가 없다.

### 신뢰의 세 기둥 (경쟁적 합의의 대체물)

| 위협 | 방어 |
|------|------|
| 위조 (남의 명의로 기록) | **서명** — 항목 전체가 시민 개인키로 서명됨 |
| 조작 (저장된 항목 고쳐 쓰기) | **서명** — 한 글자만 바꿔도 검증 실패 |
| 검열 (불리한 항목 몰래 삭제) | **상호참조 체크포인트** — 노드들이 주제 상태 요약을 서로의 로그에 엮어 두므로 대조 시 적발 |
| 이중 발언 (진영마다 다른 서명) | **분기 증명** — 같은 순번의 두 서명이 만나는 순간 암호학적 증거가 되고 집계에서 제외 |

핵심 성질: **관심이 곧 보안이다.** 어떤 주제에 관심 있는 사람이 많을수록
복제본과 상호참조가 늘어나 그 주제의 기록은 더 조작하기 어려워진다.
중요한 의제일수록 저절로 더 안전해지는, 민주주의와 정렬된 보안 모델이다.

```bash
npm run demo:weave   # 4개 노드 네트워크: 부분 복제 → 가십 수렴 → 3가지 공격 탐지 시연
```

```
src/weave/entry.js  시민별 서명 로그 항목 + 지갑(개인키는 시민 단말에만 존재)
src/weave/node.js   위브 노드: 관심 기반 복제, 가십 동기화, 체크포인트 상호참조, 결정적 집계
demo/weave-demo.js  분권 시나리오 데모 (수렴·이중발언·조작·검열 탐지)
test/weave.test.js  프로토콜 테스트 8건
```

## 3단계: 지지의 줄서기 구조 — 참여가 곧 검증

`src/weave/queue.js`는 지지를 **"그 의견 뒤에 줄을 서는 것"** 으로 재정의한다.

- **줄서기(JOIN)** = "나는 이 항목(들) 뒤에 선다"를 서명하는 것. 서명이
  앞사람 항목의 해시를 덮으므로, **줄에 서는 행위 자체가 앞사람들의 증인이
  되는 행위**다. 내 뒤에 누군가 서는 순간 내 자리는 지워질 수 없게 고정된다.
  검증 노동이 따로 필요 없다 — 참여가 곧 검증이다.
- **줄의 길이 = 현재 서 있는 사람 수.** 링크는 지울 수 없는 역사로 남고,
  떠나기(LEAVE)는 길이만 줄인다. 실시간 철회 원칙이 그대로 유지된다.
- **갈라진 줄은 저절로 아문다.** 두 사람이 동시에 줄 끝에 서면 줄이 갈라지는데
  (분산 시스템의 자연 현상), 다음 사람이 갈라진 끝들을 모두 참조하며 서면
  줄이 병합된다. 사람 수는 중복 없이 센다.
- **분기(AMEND) = 의견의 진화.** 동의하되 수정하고 싶으면 줄의 한 지점에서
  갈라져 수정안의 새 줄을 시작한다. 의견은 나무(변형 계보)로 진화하고,
  시민의 위치는 가족(원안+수정안들) 안에서 하나뿐이다 — 수정안 줄로 옮겨
  서면 원안 줄에서 자동으로 빠진다. 어떤 변형이 가장 긴 줄을 얻는지가
  실시간으로 드러난다.
- **검열 방어가 지지에 비례한다.** 줄 중간의 항목을 몰래 지우면 그 뒤에 선
  모든 사람의 참조가 허공에 뜬다 — 뒷사람 전원이 증인이다.
  **긴 줄일수록 지우기 어렵다. 지지의 크기가 곧 기록의 견고함이다.**

```bash
npm run demo:queue   # 줄서기 → 동시성 치유 → 분기 → 떠나기 → 검열 적발 시연
```

## 데모의 한계 (실제 시스템으로 가려면)

- 개인키가 서버에 보관된다 → 실제로는 클라이언트(시민 단말)가 키를 보관하고 서명해야 한다.
- 단일 노드 체인이다 → 실제로는 합의 알고리즘(PoS/BFT)을 갖춘 분산 노드가 필요하다.
- 신원 검증이 없다 → 1인 1목소리를 위한 주민 인증(영지식 증명 등) 연동이 필요하다.
- 메모리 저장이다 → 영속 저장소가 필요하다.
- 어뷰징 방어(시빌 공격, 여론 조작 봇)와 숙의 품질 장치(토론 단계, 냉각 기간 옵션)가 필요하다.
