# 보안 점검 — 위협 모델, 지수 파밍, 적대적 행위의 해석

## 1. 위협 모델 현황

| 위협 | 내용 | 상태 | 방어 수단 |
|------|------|------|-----------|
| 위조 | 남의 명의로 항목 생성 | **방어됨** | 모든 항목이 시민 개인키로 서명 (`entry.js`) |
| 조작 | 저장된 항목 고쳐 쓰기 | **방어됨** | 서명이 내용 전체를 덮음 — 한 글자만 바꿔도 검증 실패 |
| 검열 | 불리한 항목 몰래 삭제 | **방어됨** | 줄서기 뒷사람 전원이 증인(behind 참조) + 체크포인트 상호참조 |
| 이중 발언 | 진영마다 다른 서명 | **방어됨** | 같은 순번의 두 서명 = 분기 증명, 반보정 가십이 해시까지 비교하므로 반드시 만남, 집계 자동 제외 |
| 재전송 | 같은 항목 반복 주입 | **방어됨** | 해시 중복 제거 + 작성자별 순번 |
| **시빌 공격** | 1인이 계정 다수 생성해 여론 위조 | **방어됨(프로토타입)** | 자격증명 승인 모드(`identity.js`): 신뢰 발급자가 서명한 공개키만 등록부 진입. 미등록 시민의 항목은 수용 거부 |
| 스팸/저장 폭탄 | 항목 홍수로 저장·대역폭 고갈 | **방어됨(기본)** | 작성자별 속도 제한(기본 60건/분), 항목 데이터 8KB 상한, 수신 메시지 512KB 상한(초과 시 회선 차단) |
| 이클립스 공격 | 피어의 이웃을 전부 공격자로 포위 | **부분 방어** | 피어 발견으로 다중 연결 형성. 미흡: 이웃 다양성 강제, 아웃바운드 고정 슬롯 필요 |
| 전송 도청/변조 | 평문 TCP 스니핑 | **미방어** | 항목 자체는 서명돼 변조 불가하나 열람 가능. TLS/Noise 프로토콜 필요 |
| 키 도난/분실 | 단말 해킹, 폰 분실 | **미방어** | 키 순환(rotation) 항목 + 소셜 복구 필요 |
| 참여 프라이버시 | 누가 무엇을 지지했는지 공개됨 | **미방어(설계상 공개)** | 투명성과 상충. 실제 시스템: 영지식 증명으로 "유효 유권자의 1표"만 증명하는 익명 모드 필요 |
| 발급자 권력화 | 자격증명 발급자가 새 중앙이 됨 | **알려진 한계** | 복수 발급기관 임계 서명 + 가입만 관리하고 행위는 볼 수 없는 ZK 집합 증명으로 발전 |

## 2. 지수(안목·권위) 파밍 — 공격 벡터와 방어

지수가 영향력이 되는 순간 지수는 공격 대상이 된다. 특히 반대 세력이
조직적으로 지수를 조작(파밍)하는 시나리오:

### 공격 벡터

1. **시빌 링 펌핑**: 가짜 계정들이 서로의 의견에 줄 서서 안목을 상호 부양.
2. **어서리티 암살**: 표적 시민이 지지한 의견에 반대를 몰아 헛발질 감점을
   유도, 그의 안목(과 권위 기여)을 깎는다.
3. **선점 봇**: 새로 올라오는 모든 의견에 기계적으로 1등 줄서기 —
   어떤 것이든 뜨면 안목을 챙기는 전략.
4. **셀프 이슈 양산**: 자기 이슈를 대량 공표하고 자기 진영이 초기 줄을 채움.

### 구현된 방어

- **신원 비용 (근본 방어)**: 시빌 링의 전제는 계정 무한 생성이다.
  자격증명 승인 모드에서 계정 = 실존 유권자이므로, 링의 크기는 실제
  사람 수로 제한된다. *지수 파밍의 1차 방어는 지수 설계가 아니라 신원이다.*
- **평균 희석 (선점 봇 방어)**: 안목은 참여한 줄들의 평균이다(특허 청구항 8).
  모든 의견에 줄 서는 봇은 실패한 줄들의 0점·감점이 쌓여 평균이 무너진다.
  선별 없는 참여는 수학적으로 손해다.
- **격차 기반 감점 (어서리티 암살의 비용)**: 헛발질 감점은 반대가 지지를
  "초과한 격차"만큼만 발생한다. 표적의 안목을 깎으려면 반대 세력이 실제로
  다수가 되어야 한다 — 진짜 신원으로 다수를 만들었다면 그것은 공격이 아니라
  그냥 민주주의다. 소수의 반대 몰이는 '경합'을 만들 뿐 감점을 만들지 못한다.
- **기본 1표 불가침**: 감점이 아무리 커도 권위 기여는 1 밑으로 내려가지
  않는다. 지수 공격으로 참정권을 박탈할 수 없다.
- **실시간 회복 (레그 없음)**: 감점은 저장되지 않고 매번 재계산된다.
  조작 세력이 반대 줄에서 빠지면(또는 조직 동원이 걷히면) 감점도 사라진다.
  일시적 몰이의 효과는 일시적이다.
- **영구 감사 기록**: 모든 줄서기는 서명된 항목이다. "이 계정들은 항상
  같이 움직인다", "이 반대는 표적의 지지 직후에만 발생한다" 같은 파밍
  패턴이 누구나 검증 가능한 데이터로 남는다. 코드가 처벌하지 않아도
  증거가 축적된다.

### 검토했으나 미구현 (다음 단계 후보)

- **안목 가중 감점**: 감점 크기를 반대자들의 안목으로 가중 — 안목 없는
  계정 100개의 반대는 안목 있는 시민 3명의 반대보다 가볍게. (순환 정의를
  피하려면 2-패스 계산 필요)
- **동행 할인(co-occurrence discount)**: 항상 같은 조합으로 줄 서는
  집단의 상호 기여를 감쇠 — 링 파밍의 수익률을 직접 떨어뜨린다.
- **시간 감쇠**: 오래된 안목의 반감기 — 과거의 영광으로 영원히 무거운
  목소리를 갖는 것을 방지 (이 역시 "레그 제거"의 연장).
- **분야별 안목 분리**: 경제에서 쌓은 안목이 외교 사안에서 그대로
  가중되지 않도록 도메인별 지수 분리.

## 3. 적대적 행위의 해석과 용인 기준

민주주의 시스템에서 "적대적 행위"의 정의는 기술 문제이기 전에 정치철학
문제다. 반대는 공격이 아니라 이 시스템의 존재 이유다. 기준선:

| 행위 | 해석 | 처리 |
|------|------|------|
| 조직적 반대·집단 동원 | **적법한 정치** — 진짜 신원들이 규칙 안에서 다수를 만드는 것은 곧 민주주의 그 자체다. 마음에 들지 않는 결과와 공격을 혼동하지 말 것 | **용인.** 시스템은 개입하지 않는다 |
| 판단 실수 (헛발질) | **부정이 아님** — 틀리는 것은 참여의 일부 | 감점은 즉시 회복 가능, 참정권 불변 |
| 전략적 줄서기 (지수 게이밍) | **회색지대** — 진짜 신원이 자기 이익을 위해 규칙을 최대한 활용하는 것 | **행위 금지 대신 이득 제거**: 평균 희석·격차 감점으로 수익률을 낮추고, 패턴은 공개 기록으로 남긴다. 코드가 판사가 되지 말고 증거를 만들게 하라 |
| 시빌 (신원 위조) | **불용** — 1인 1목소리의 직접 위반. 정치가 아니라 사기 | 기술적 차단 (자격증명 승인) |
| 이중 발언 | **불용** — 암호학적으로 증명되는 기만 | 자동 증명·집계 제외 (사회적 판단 불필요 — 수학이 판정) |
| 기록 조작·검열 | **불용** — 공론장 자체의 파괴 | 서명·상호참조·증인 구조로 자동 적발 |
| 자원 고갈 공격 (스팸·폭탄) | **불용** — 발언이 아니라 방해 | 속도·크기 제한 (발언 내용은 심사하지 않고 물리량만 제한) |

**설계 원칙 요약**: ① 내용은 절대 심사하지 않는다 — 어떤 주장이든 줄을 설
수 있다. ② 신원과 물리량(속도·크기)만 기술적으로 통제한다. ③ 수학적으로
증명 가능한 기만(이중 발언, 위조, 조작)만 자동 배제한다. ④ 나머지 모든
전략적 행동은 유인 설계(평균·격차·회복)로 다루고, 최종 판단은 영구 기록을
보는 시민들에게 남긴다.
